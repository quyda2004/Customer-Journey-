# Customer Journey

This **Customer Journey** project is a hands-on opportunity to demonstrate end-to-end skills in **data analysis**, **data model design**, **metadata & data quality assurance**, **insight generation for decision-making**, and **predictive modeling**. The goal is to build practical experience in solving common data challenges in the banking industry.

## Project Overview

The project focuses on the **Customer Onboarding Journey**—the process of welcoming new customers into a bank’s digital ecosystem. I simulate a real-world scenario where the bank applies a digital onboarding flow using **eKYC (Electronic Know Your Customer)** technology.

This journey covers the full lifecycle—from introducing services to customers, creating a digital account, and conducting transactions, to **post-onboarding monitoring** that tracks transaction behavior. The ultimate objective is to assess potential fraud risk, strengthen fraud controls, and support the development of an effective fraud prediction model.

## Stages

### [CASE 1 — Data Model Design](https://github.com/quyda2004/Customer-Journey-/tree/main/CASE%201%20-%20DATA%20MODEL%20DESIGN)
In this stage, I identify the core onboarding flow and define key entities, attributes, and relationships. I also establish foundational standards for data collection and storage. The main focus is to design a scalable and consistent data model that supports long-term analytics and downstream use cases.

### [CASE 2 — Metadata Management & Data Quality](https://github.com/quyda2004/Customer-Journey-/tree/main/CASE%202%20-%20METADATA%20MANAGEMENT%20AND%20DATA%20QUALITY)
In this stage, I explore how to manage metadata and enforce data quality. The focus is on identifying critical data attributes, validating integrity, and ensuring data is accurate, consistent, and ready for analytics and modeling.

### [CASE 3 — Reporting & Analytics](https://github.com/quyda2004/Customer-Journey-/tree/main/CASE%203%20-%20REPORTING%20AND%20ANALYTICS)
In this stage, I build reporting and analytics on top of the onboarding journey. I define key success metrics (e.g., funnel performance, activation, and behavior indicators) and use analysis/visualization to generate insights that help improve onboarding effectiveness.

### [CASE 4 — Fraud Prediction](https://github.com/quyda2004/Customer-Journey-/tree/main/CASE%204%20-%20FRAUD%20PREDICTION)
In this stage, I evaluate the onboarding verification outcomes and post-eKYC behavior—from account opening and the first transaction to the point where the system determines a customer’s **Fraud Type**. Finally, I develop a fraud prediction model to detect high-risk cases early, enabling proactive prevention and improving overall onboarding security.
